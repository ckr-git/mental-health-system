## 2. API 请求生命周期

```mermaid
sequenceDiagram
    participant C as 前端组件
    participant S as Pinia Store
    participant I as Axios 拦截器
    participant N as 网络
    participant CF as CORS Filter
    participant JF as JWT Filter
    participant SC as Security Chain
    participant Ctrl as Controller
    participant Svc as Service
    participant MP as MyBatis-Plus
    participant DB as MySQL

    C->>S: 调用 action
    S->>I: 发起请求

    Note over I: 请求拦截器
    I->>I: 注入 Authorization: Bearer <token>
    I->>I: 设置 Content-Type
    I->>N: 发送请求

    N->>CF: CORS 检查
    CF->>JF: Token 提取与校验

    alt Token 有效
        JF->>JF: 解析 userId, role
        JF->>SC: 设置 SecurityContext
        SC->>SC: 检查角色权限

        alt 权限通过
            SC->>Ctrl: 路由到 Controller
            Ctrl->>Ctrl: 参数校验
            Ctrl->>Svc: 调用 Service
            Svc->>MP: 数据操作
            MP->>DB: SQL 执行
            DB-->>MP: 结果集
            MP-->>Svc: Entity/List
            Svc-->>Ctrl: 业务结果
            Ctrl-->>N: Result.success(data)
        else 权限不足
            SC-->>N: 403 Forbidden
        end
    else Token 无效/过期
        JF-->>N: 401 Unauthorized
    end

    N-->>I: HTTP Response

    Note over I: 响应拦截器
    alt 状态码 200
        I-->>S: 返回 data
        S-->>C: 更新状态
    else 状态码 401
        I->>I: 清除 Token
        I->>I: 跳转登录页
    else 状态码 其他
        I->>I: ElMessage.error 提示
        I-->>S: 抛出异常
    end
```

## 3. WebSocket 实时通信

```mermaid
sequenceDiagram
    participant P as 患者前端
    participant WS as WebSocket Server
    participant DB as MySQL
    participant D as 医生前端

    Note over P,D: 建立连接
    P->>WS: CONNECT /ws/chat (SockJS)
    WS-->>P: CONNECTED
    P->>WS: SUBSCRIBE /user/{userId}/queue/messages
    P->>WS: SUBSCRIBE /user/{userId}/queue/notifications

    D->>WS: CONNECT /ws/chat (SockJS)
    WS-->>D: CONNECTED
    D->>WS: SUBSCRIBE /user/{doctorId}/queue/messages
    D->>WS: SUBSCRIBE /user/{doctorId}/queue/notifications

    Note over P,D: 消息收发
    P->>WS: SEND /app/chat.send
    Note right of P: {senderId, receiverId,<br/>content, messageType}
    WS->>DB: 保存 ChatMessage
    WS->>D: MESSAGE /user/{doctorId}/queue/messages
    Note right of D: 实时收到消息

    D->>WS: SEND /app/chat.send
    WS->>DB: 保存 ChatMessage
    WS->>P: MESSAGE /user/{userId}/queue/messages

    Note over P,D: 系统通知
    WS->>P: MESSAGE /user/{userId}/queue/notifications
    Note right of WS: {type: "APPOINTMENT_CONFIRMED",<br/>title: "预约已确认"}
    WS->>D: MESSAGE /user/{doctorId}/queue/notifications
    Note right of WS: {type: "CRISIS_ALERT",<br/>title: "患者危机预警"}
```
